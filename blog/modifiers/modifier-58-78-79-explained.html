Modifiers 58, 78, and 79 Explained for Surgeons and Coders

Short answer: modifier 58 is for a staged or more extensive planned procedure during the global period, modifier 78 is for an unplanned return to the operating room for a related procedure, and modifier 79 is for an unrelated procedure by the same physician during the global period. These modifiers live in the same neighborhood, but they do not mean the same thing.

58 = staged, planned, or more extensive related procedure
78 = unplanned return to OR for a related procedure
79 = unrelated procedure during the global period

Modifier 58 explained

Use modifier 58 when the follow-up procedure is part of the expected therapeutic pathway, was prospectively staged, or is more extensive than the original procedure. This is not the modifier for complications. It is the modifier for planned escalation or a logical next step.

  • Diagnostic procedure followed by definitive therapeutic procedure
  • Limited procedure followed by more extensive procedure
  • Staged return that was anticipated as part of treatment

Modifier 78 explained

Use modifier 78 when the patient must return to the operating room unexpectedly for a related problem during the global period. Think bleeding, infection, dehiscence, or other complication management that requires an operative return.

Important: modifier 78 is not for a clinic visit, bedside dressing change, or routine post-op management. It is for an unplanned related return to the operating room or qualifying procedure room.

Modifier 79 explained

Use modifier 79 when the same physician performs a completely unrelated procedure during the global period of the first surgery. The second procedure is not part of the original post-op course and should stand on its own.

  • Original surgery: laparoscopic cholecystectomy
  • Later unrelated surgery during that global: incision and drainage of unrelated abscess
  • That is a potential modifier 79 scenario, assuming documentation supports unrelatedness

Common coding mistakes

  • Using modifier 58 for a complication return that belongs under 78
  • Using modifier 78 when the second procedure was actually unrelated and belongs under 79
  • Forgetting that 58 and 79 often reset the global, while 78 usually does not
  • Not documenting why the second procedure was staged, related, or unrelated

How to decide fast

Ask three questions

  • Was the second procedure planned or a logical staged escalation? If yes, think 58.
  • Was the second procedure an unplanned return for a related complication? If yes, think 78.
  • Was the second procedure unrelated to the original surgery? If yes, think 79.

Documentation tips

  • For modifier 58, document the staged nature or therapeutic progression
  • For modifier 78, document the complication and return-to-OR necessity
  • For modifier 79, explicitly state why the second procedure is unrelated

FAQ

Does modifier 78 start a new global period?

Usually no. Modifier 78 generally keeps the case inside the original global period.

Does modifier 79 start a new global period?

Usually yes, because the second procedure is unrelated.

Can modifier 58 be used for a complication?

Usually no. If the return is for a complication, 78 is more likely the correct logic.
