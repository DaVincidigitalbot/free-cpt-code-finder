# FreeCPTCodeFinder Blog Publishing Checklist

## Repeatable article template
Use this structure every time.

1. SEO target
- Primary keyword
- 2 to 4 secondary keywords
- Search intent: informational / comparison / coding rule / reimbursement
- URL slug

2. Metadata
- Title tag under about 60 characters when possible
- Meta description 140 to 160 characters
- Canonical URL
- OG title and description

3. Article skeleton
- H1 with exact query intent
- Opening answer in first 120 words
- Quick answer table or summary box
- Main sections explaining when to use each code or rule
- Common mistakes / denial traps section
- FAQ section with long-tail variants
- CTA back to Free CPT Code Finder
- Related guides section

4. Internal linking minimums
- 2 same-cluster links
- 1 adjacent-cluster link
- 1 home page tool link

5. Quality controls
- Specific codes and rules checked
- No vague filler paragraphs
- No duplicate intent versus an existing page
- No unsupported claims
- No fake dates or fabricated policy changes

## Clean daily publishing workflow
### Before writing
- Confirm the page does not already exist
- Confirm the keyword target is not cannibalizing an existing post
- Pick the adjacent pages to interlink before drafting
- Decide the primary CTA

### During writing
- Answer the query immediately
- Use short sections and tables
- Include exact code sets, modifier rules, or diagnosis buckets
- Write like a clinician-coder, not a content farm
- Keep tone direct and practical

### Before publishing
- Title matches search intent
- Meta description is specific
- H1 is clean
- Breadcrumb is correct
- Related guides are relevant
- Internal links work
- CTA to tool is present
- Font/colors match site
- Mobile layout looks sane

### After publishing
- Add link to /blog/ index if needed
- Update BLOG-QUEUE.md to published
- Add to sitemap if workflow requires it
- Check page loads live
- Spot-check one desktop and one mobile render
- Record publish date

## Red flags
- Query already covered by a stronger page
- Article too broad to rank for anything
- Generic intro that wastes the first paragraph
- No examples
- No internal links
- Writing for “health readers” instead of coders/clinicians
- Thin article written just to hit a daily quota

## Fast scoring rubric before publish
- Intent clarity: pass / fail
- Accuracy: pass / fail
- Internal linking: pass / fail
- CTA present: pass / fail
- Search-title strength: pass / fail
- Actually useful: pass / fail

If any item fails, do not publish that day. Missing a day is better than posting junk.
