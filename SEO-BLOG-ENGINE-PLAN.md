# FreeCPTCodeFinder Blog SEO Engine Plan

## Objective
Turn the blog into a search acquisition system that captures high-intent medical coding traffic, routes readers into the CPT finder, and strengthens topical authority around CPT, modifiers, ICD-10, RVUs, and coding workflow.

## Hard rules
- Publish useful posts, not AI sludge.
- Every post must target a real query with buyer or practitioner intent.
- Every post must link to the CPT finder and 2 to 4 relevant internal pages.
- Accuracy beats volume. Thin or wrong content will hurt more than help.
- Update winners quarterly instead of endlessly spraying new weak pages.

## Current asset base
- Existing blog categories:
  - CPT guides: 30
  - Modifiers: 8
  - ICD-10: 4
  - RVU: 5
- Existing strengths:
  - Strong long-tail surgical coding coverage
  - Good article volume already live
  - Direct commercial bridge to tool usage
- Current weaknesses:
  - No explicit cluster map
  - Internal linking appears inconsistent across topics
  - No visible editorial scoring or priority system
  - Some content is surgeon-centered when broader coder traffic exists

## SEO strategy
Build around 5 content clusters:

### 1. CPT procedure coding cluster
Purpose: Capture high-intent searchers looking for exact procedure code help.

Core page types:
- "CPT code for [procedure]"
- "[procedure] CPT code"
- "open vs laparoscopic [procedure] CPT"
- "[procedure] CPT with modifier"

Primary internal link targets:
- Home page tool
- Relevant code pages
- Related modifier pages
- Related ICD-10 diagnosis pages

Best existing examples:
- appendectomy
- central line
- colon resection
- thyroid surgery
- ventral hernia repair

### 2. Modifier cluster
Purpose: Own denial-prevention and payment optimization intent.

Core page types:
- "modifier 25 explained"
- "modifier 25 vs 57"
- "modifier 51 vs 59"
- "modifier 58 78 79 explained"
- specialty-specific modifier use cases

Primary internal link targets:
- global period pages
- multiple procedure pages
- office procedure pages
- CPT procedure guides where modifier choice matters

### 3. ICD-10 cluster
Purpose: Capture diagnosis-to-procedure coding searches and broaden authority beyond CPT.

Core page types:
- "ICD-10 coding for [diagnosis]"
- "common ICD-10 mistakes"
- specialty diagnosis bundles
- documentation specificity pages

Primary internal link targets:
- related CPT guides
- documentation pages
- diagnosis-specific blog pages

### 4. RVU and reimbursement cluster
Purpose: Capture compensation and business-of-medicine traffic.

Core page types:
- "work RVU for [procedure]"
- "highest RVU general surgery procedures"
- "how surgeons get paid"
- "increase surgical RVUs legally"
- CPT-to-wRVU lookup intent

Primary internal link targets:
- specific procedure guides
- modifier 22
- documentation pages
- home page calculator/tool

### 5. General coding workflow cluster
Purpose: Capture broader educational traffic and create top-of-funnel entry points.

Core page types:
- "how to learn CPT coding"
- "most common CPT codes"
- "time-based E/M billing"

- "documentation for maximum reimbursement"

Primary internal link targets:
- cluster hubs
- procedure pages
- modifier pages
- tool home page

## Internal linking rules
Every new post should include:
- 1 link to home page tool near top or middle CTA
- 2 links to same-cluster posts
- 1 link to adjacent-cluster content
- 1 link to the most relevant procedure or code page when applicable

Preferred internal link patterns:
- Procedure post → modifier page + ICD-10 page + RVU page
- Modifier post → global period page + procedure examples + tool
- ICD-10 post → diagnosis-relevant procedure pages + documentation page
- RVU post → modifier 22 + multiple procedures + specific surgical examples
- Broad educational post → best-performing long-tail posts

## On-page SEO standards
For every article:
- Title: exact or near-exact query target, front-loaded
- Meta description: 140 to 160 chars, practical and specific
- H1: match search intent clearly
- First 120 words: answer query directly
- Include comparison tables where useful
- Use FAQ section for long-tail variants
- Use breadcrumb links
- Add related guides section with real relevance, not random stuffing
- Include one primary CTA back to FreeCPTCodeFinder tool

## Recommended publishing cadence
- 1 high-quality post per day is fine
- 5 to 7 per week is enough
- Each week should include:
  - 3 CPT procedure posts
  - 1 modifier post
  - 1 ICD-10 or RVU post
  - 1 broader workflow or E/M post
- Every 2 weeks: refresh 2 older winners with tighter intros, better links, and updated examples

2. Show the code set or rule table fast
3. Explain when each code applies
4. Show common mistakes and denial traps
5. Link to related modifiers and ICD-10 pages
6. End with CTA to Free CPT Code Finder

## What not to do
- No generic health content
- No patient-facing fluff
- No duplicate pages targeting the same exact intent without a clear reason
- No fake freshness posts with no coding substance
- No stuffing city names or physician keywords into coding pages

## Operating recommendation
Do not just post daily at random.
Run a repeating weekly pattern:
- Monday: high-intent CPT post
- Tuesday: modifier post
- Wednesday: ICD-10 post
- Thursday: high-intent CPT or E/M post
- Friday: RVU/reimbursement post
- Saturday: broad educational or comparison post
- Sunday: update one old winner or publish specialty expansion

That keeps the blog coherent, internally linked, and a lot less sloppy than a blind daily dump.
